# JavaQueue  
Test Task: Working with Queues and Multithreading  

Выполненное тестовое задание:    
Отправка данных в GIS систему в конечную точку Rest API Json  

Описание:  
Создать метод который будет принимать данные для передачи в GIS установить лимит на количество одновременных сессий на добавление новых элементов не больше 5 за 1 минуту  
так же добавить функцию которая будет сохранять не отправленные данные и отправлять их чуть позже.  

Решение:
Я использую 2 очереди используя ArrayBlockingQueue одна для очередей отправки, другая для сохранение не добавленых элементов.  
RateLimiter -> tryAcquire хранит и проверяет сколько запросов было сделано за 1 минуту и сбрасывает счетчик
